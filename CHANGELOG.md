- **Expose refuse_unreliable end-to-end; stop the CLI crowning gate-flagged policies**: diagnostics/README.md claimed estimators "refuse estimation (return NaN)" automatically while `refuse_unreliable` defaulted to False and the DR/stacked factory builders silently dropped the key. The flag now flows from `analyze_dataset(estimator_config={"refuse_unreliable": True})` to CalibratedIPS, raw-ips, and every DR estimator's internal `ips_estimator` (dr-cpo/mrdr/tmle and stacked-dr components); the README's "Diagnostic Gates" section and design principle #3 now state the real default (warn loudly, estimate anyway; NaN only on request) and document that the Hill tail index is a `weight_status` input, not a NaN-refusal gate. Per-policy gate outcomes are recorded in `result.metadata["reliability_gates"]` (`flagged`/`refused`/`reasons`) regardless of the flag. The CLI previously printed "🏆 Best policy" from the raw argmax even when that policy was flagged UNRELIABLE (on the bundled arena sample the adversarial 'unhelpful' policy won the trophy with the warning scrolled away): `cje analyze` now consults the reliability gates and per-policy statuses — a flagged argmax prints "⚠️ Best by point estimate: X (UNRELIABLE — see diagnostics)" followed by the best RELIABLE policy when one exists, and all-NaN results say so instead of crowning anything.
- **Implement the paper's judge-space overlap gate and wire in the coverage badge**: the library graded overlap only via action-space `E[sqrt(w)]` ("good" at 0.50) while the paper gates on the binned judge-space Bhattacharyya affinity A_B >= 0.85; the declared `aessf_sigmaS`/`bc_sigmaS` fields were never populated (CF-bits vestiges) and the README claimed nonexistent "σ(S) floors". New `bhattacharyya_judge_space(...)` computes the paper's binned A_B (fresh-draw S vs logged S when draws are available — DR does this automatically — else the raw-weight-weighted vs unweighted logged-S histogram); `CalibratedIPS` populates `IPSDiagnostics.bc_sigmaS_per_policy` and folds it into statuses via the canonical gates (>= 0.85 pass, < 0.5 severe); `OverlapMetrics.bc_sigmaS`/`aessf_sigmaS` (= A_B², the σ(S) structural ESS ceiling) are populated when judge scores are passed; `hellinger_affinity` is documented as the distinct action-space quantity (unchanged, not silently redefined; the unused `aessf_sigmaS_lcb` field was removed). Separately, `boundary_card` (the paper's REFUSE-LEVEL coverage badge, out-of-range >= 5%) had zero callers and was unexported while `CJEDiagnostics` hardcoded `coverage_risk="low"` so `refuse_level_claims` could never fire: `boundary_card`/`BoundaryCard` are now exported, `CalibratedIPS` computes a per-policy card against the oracle calibration S-range (attached to `diagnostics.boundary_cards` + `result.metadata`, copied into DR diagnostics, loud warning on REFUSE-LEVEL), and `CJEDiagnostics.from_ips_diagnostics` derives `coverage_risk` from the cards (REFUSE-LEVEL => "critical" => `refuse_level_claims=True`; no cards => "unknown", never a hardcoded "low", and `can_make_level_claims` stays False until coverage is actually checked).
- **Compute the paper's TTC gate during estimation**: the paper's headline diagnostic (Target-Typicality Coverage; gate TTC >= 0.70 before trusting logged data) existed as `compute_ttc` but had zero callers in the pipeline AND implemented a different statistic (total sequence surprisal at 0.8 target mass instead of the paper's mean per-token surprisal at q_0.9 — total NLL turns the "typical" region into a short-response region). `compute_ttc`/`compute_cle_diagnostics` now default to `target_typical_mass=0.9` and accept `token_counts` for the per-token construction; `CalibratedIPS` computes TTC per policy at estimation time (using a `response_token_count` metadata field when present, else response length in characters as a documented proxy, recorded in `result.metadata["ttc_diagnostics"]`), populates the new `IPSDiagnostics.ttc_per_policy` field (copied into `DRDiagnostics`, along with the previously dropped per-policy status/tail/Hellinger fields), folds TTC into `weight_status`/`status_per_policy` via the canonical gates (WARNING < 0.70, CRITICAL < 0.30), warns loudly when the gate fails, and surfaces "Worst TTC" in `summary()`, `validate()`, and CSV export. Fixed the `CLEDiagnostics` docstring claiming "alpha ≈ 0.8 by construction" and the mislabeling of `E[sqrt(w)]` as a "surrogate space" quantity.
- **One canonical ESS ladder + honest tail-index failures**: ESS thresholds were mutually inconsistent across six surfaces (weights.py 0.01/0.1, the CalibratedIPS refusal gate 0.30, display.py 0.5/0.2, weight dashboards 0.5/0.2/0.1, and two contradictory ladders inside diagnostics/README.md) — a policy at ESS 0.25 could read GOOD, "UNRELIABLE ESTIMATE", and "Good — usable with caution" in the same run. All surfaces now import one ladder from the new `cje/diagnostics/gates.py` (`ess_status`: >= 0.30 GOOD — the paper's ship-gate — / >= 0.10 WARNING / else CRITICAL), which also centralizes the TTC, judge-space Bhattacharyya, tail-index, and coverage-badge thresholds. `hill_tail_index` now returns NaN on estimation failure (too few samples, >~95% exact-zero weights, numerical issues) instead of +inf, which every downstream check previously read as the healthiest possible tail; +inf is reserved for genuinely uniform tails. NaN tails are treated as *unknown* downstream (`tail_status` escalates to WARNING when ESS is also below the ship-gate; `worst_tail_index` no longer launders failures through `min(..., default=inf)`; `validate()` reports the failure; `analyze_diagnostics` counts `tail_unknown` separately from `heavy_tail`; display helper renders "n/a"). Regression tests in `cje/tests/test_refusal_gates.py`.
- **Keep every oracle (judge, oracle) pair when combining sources**: `_combine_oracle_sources` used to dedupe by prompt_id alone, so K policies' fresh-draw labels for the same prompt collapsed to the last-iterated policy's pair while `n_from_fresh` counted every overwrite. Oracle labels attach to responses, not prompts: all pairs now enter calibration (only true duplicates — same source + prompt + identical judge & oracle values — are deduped), per-source counts reflect what actually enters, the cross-source conflict warning fires unconditionally, and the obsolete `priority_order` metadata key is gone. Also fixed `CalibratedIPS.fit`'s duplicate-prompt_id branch, where an unbound `ds_idx` raised an `UnboundLocalError` that a blanket except swallowed — silently skipping the fold-based OOF fallback it was supposed to fall through to.
- **Fix calibration misalignment and fabricated fallbacks**: integer (index) `oracle_mask` inputs to `JudgeCalibrator.fit_transform`/`fit_cv` are now argsorted with oracle labels reordered identically (unsorted indices used to silently misalign every (score, label) pair; verified RMSE 0.032 → 0.302) and duplicate indices are rejected; `predict_oof` raises for fold ids without a fitted model instead of zero-filling predictions; `FlexibleCalibrator` raises `RuntimeError` instead of fabricating constant-0.5 or mean-of-judge-score rewards when no models are fitted; and monotone calibration warns loudly when the fitted mapping collapses to a constant while oracle labels vary (anti-correlated/inverted judge scale), which previously erased all policy differences silently.
- **Harden data loading and validation against silent corruption**: `Sample` now rejects NaN/±inf logprobs (previously they passed validation and silently zeroed or exploded importance weights; use `None` for missing); `load_fresh_draws_auto` propagates per-record parse errors with file/line context instead of swallowing them and misreporting `FileNotFoundError`; `DatasetLoader` no longer hash-replaces falsy-but-valid prompt_ids (0, "") and coerces integer ids via `str()` (matching fresh-draw loading so joins work), replaces `print()` skips with counted `logger.warning`s and raises when every record was skipped; `validate_cje_data` scans the reward field over the same window as judge/oracle checks (was: only `data[0]`) and reports the missing fraction; `add_rewards_to_existing_data` keys the judge-score collection and reward join on record index so duplicate prompt_ids each get f(their own judge score) instead of all inheriting the last record's reward.
- **Fail loudly on silent interface downgrades**: `analyze_dataset` now raises when `fresh_draws_data` is combined with `logged_data_path` (the in-memory draws were silently ignored — no DR, metadata claimed `has_fresh_draws=False`); auto mode raises `FileNotFoundError` when `fresh_draws_dir` does not exist instead of silently downgrading DR→IPS; and zero-oracle Direct runs emit a prominent warning and are labeled `method="naive_direct"` instead of `"calibrated_direct"` (uncalibrated judge-score means were previously indistinguishable from calibrated results).
- **Fix CalibratedIPS influence function understating the SE ~2x**: (a) the Hajek ratio IF included an extra `−ψ(w−mean_w)` term that double-counted the ratio-denominator correction (the delta method gives `φ = w(R−ψ)/mean_w`); (b) at 100% oracle coverage the IF was built from smoothed OOF calibrator predictions while the estimate consumed raw oracle labels, deleting the reward-noise term from the variance. Discovered by the new Monte Carlo coverage harness (reported-SE/empirical-SD ratio was 0.44-0.63 at full oracle coverage; now ~1.0).
- **Add a Monte Carlo ground-truth coverage harness** (`cje/tests/test_mc_coverage.py`): synthetic DGP with a known policy value; fast CI layer asserts point-estimate bias and reported-SE-vs-empirical-SD bands for calibrated-ips and dr-cpo (catching any resurgence of the OUA ×K, MC double-count, or IF-formula bugs), plus a slow 300-replicate 95% CI coverage layer across direct/calibrated-ips/dr-cpo/tmle/stacked-dr.

- **Unify DR-family standard errors and stop double-counting Monte Carlo variance**:
  - DR-CPO no longer adds a separate fresh-draw MC variance term to its SE. The influence functions are built from per-prompt means of the fresh-draw predictions, so the (cluster-robust) IF variance already contains the within-prompt MC noise; re-adding `mc_var` in quadrature inflated SEs by up to ~sqrt(2)x when M=1. MC numbers remain available as diagnostics in `metadata["mc_variance_diagnostics"]` (per-prompt variances now use ddof=1). Reported CIs narrow by design.
  - TMLE and MRDR previously reported naive IID SEs (`np.std/sqrt(n)`) with no cluster-robust correction and no `degrees_of_freedom` metadata (so their CIs silently used z instead of t) — a different basis from DR-CPO. All three DR estimators now share one SE helper (`DREstimator._compute_policy_se`): cluster-robust IF SE on outcome folds + delete-one-fold oracle-jackknife variance (added exactly once; `_apply_oua_jackknife` no longer re-adds it), with t-based df metadata and a per-policy SE decomposition in `metadata["se_components"]`.
  - StackedDR: removed the MC-aware rank-1 term from the weight objective and the `mc_var` composition from the stacked SE. Only dr-cpo ever reported MC diagnostics, so the penalty asymmetrically inflated dr-cpo's diagonal in the weight QP and tilted weights toward tmle/mrdr for reasons unrelated to efficiency; the stacked IF already contains the MC noise. `include_mc_in_objective`/`mc_lambda` kwargs are accepted with a `DeprecationWarning` and ignored. Aggregated MC variance is diagnostics-only.
  - `compute_robust_inference` (public diagnostics API): bootstrap/cluster CIs were centered at ~0 (mean-centered influence functions, returned unshifted) and never contained the estimate; they are now re-centered on each point estimate. The raw `data=` path used a constant placeholder statistic (SE ~1e-16, zero-width CI, p=0.0) and now raises `ValueError` directing users to pass `influence_functions`.
  - Regression tests in `cje/tests/test_inference_unification.py` pin: exact SE composition (IF + oracle only) with MC as metadata; all three DR estimators routing through the shared helper with t-based df; cluster-robust (not naive) IF SEs; equal stacking weights for identical IFs regardless of reported MC diagnostics; and CI-contains-estimate for every `compute_robust_inference` method.
- **Fix OUA jackknife variance understated by a factor of K** in `base_estimator._apply_oua_jackknife` and `stacking._apply_stacked_oua` (mean over folds instead of the delete-one-fold jackknife sum, paper Alg. 6). With the default K=5, the calibration-uncertainty contribution to reported standard errors was ~2.24x too small for calibrated-ips, calibrated direct (jackknife path), TMLE, MRDR, and stacked-dr. All OUA sites now share one `oracle_jackknife_variance()` helper. Reported CIs widen by design.
- **Fix `calibration_data_path` replacing the evaluation dataset**: in IPS/DR mode the estimator was built from the calibration (or combined-oracle) pool — whose fabricated logprobs made every policy's estimate identical with a perfect ESS — instead of the logged data. The calibration pool is now used only to fit the calibrator, whose rewards are applied to the logged evaluation dataset before estimation. Regression tests pin distinct per-policy estimates and logged-length influence functions.
- **Fix TMLE collapsing to the untargeted plug-in**: the fluctuation was solved on logged data but never propagated to fresh draws, so the solved score equation cancelled the IPS correction exactly and `TMLEEstimator` returned the direct-method plug-in with a DR-type SE (no double robustness). The targeting step now uses a score-indexed clever covariate h(S)=E[w|S] (fit per fold; exact for S-calibrated weights) and applies the targeted outcome model to fresh draws per draw. Regression tests pin double robustness under deliberate outcome-model misspecification and first-order agreement with DR-CPO.
- **Fix OUA jackknife predictions for two-stage calibrators**: leave-one-fold reward predictions in `CalibratedIPS`, `DREstimator`, and `CalibratedDirectEstimator` fed raw judge scores to per-fold isotonic models that expect the ECDF rank index, corrupting jackknife replicates whenever `two_stage` calibration was active (auto-selectable by default; always active with covariates). All jackknife paths now route through `reward_calibrator.predict_oof(...)`, and `CalibratedIPS` gained covariate support in its jackknife.
