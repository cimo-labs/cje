# CJE Interface

The high-level entry points: `analyze_dataset()` and the `cje` CLI. One mode — calibrated Direct evaluation of policies from judge-scored fresh draws.

For an end-to-end operational workflow (audits, drift response, label budgeting), see [`PLAYBOOK.md`](../../PLAYBOOK.md).

## Quick Start

```python
from cje import analyze_dataset

results = analyze_dataset(fresh_draws_dir="responses/")

for i, policy in enumerate(results.metadata["target_policies"]):
    print(f"{policy}: {results.estimates[i]:.3f} ± {1.96*results.standard_errors[i]:.3f}")
```

### What are fresh draws?

New responses generated by each policy you want to compare, evaluated by your judge, with oracle labels on the slice you can afford.

**Format:** one JSONL file per policy in `fresh_draws_dir`; the policy name is inferred from the filename (`responses/clone_responses.jsonl` → policy `"clone"`). Each record needs `prompt_id` + `judge_score`; `oracle_label` is optional per record. See [`cje/data/README.md`](../data/README.md) for the full schema.

## Common Workflows

### In-memory data (no file I/O)

CJE wants at least 10 oracle-labeled samples pooled across policies (2 per calibration fold for the default 5 folds; with 4–9 labels the fold count auto-reduces with a warning and CIs get noisier, below 4 it raises) — in practice, label 5–25% of your data. The labels can all come from one policy: CJE learns the judge→oracle mapping once and applies it to every policy (the transport audit checks that reuse):

```python
from cje import analyze_dataset

# policy_a carries the oracle slice; policy_b is judge-only
labeled = [(0.62, 0.55), (0.68, 0.60), (0.72, 0.70), (0.76, 0.74), (0.79, 0.75),
           (0.83, 0.80), (0.85, 0.90), (0.88, 0.92), (0.91, 0.88), (0.95, 0.97)]
judge_only = [0.70, 0.74, 0.75, 0.78, 0.81, 0.83, 0.86, 0.90, 0.93, 0.94]

results = analyze_dataset(
    fresh_draws_data={
        "policy_a": [
            {"prompt_id": f"q{i:02d}", "judge_score": s, "oracle_label": y}
            for i, (s, y) in enumerate(labeled)
        ],
        "policy_b": [
            {"prompt_id": f"q{i:02d}", "judge_score": s}
            for i, s in enumerate(judge_only)
        ],
    }
)
```

For a single sample of judge scores + a partial oracle slice as plain numpy arrays, use the array API instead: `from cje import calibrated_mean_ci` (see the root README).

### Dedicated calibration data

Learn the judge→oracle mapping from a separate file — a curated oracle set, or your old 0.3.x logged data (judge + oracle pairs; logprob fields are ignored). A minimal `{prompt_id, judge_score, oracle_label}` file works:

```python
results = analyze_dataset(
    fresh_draws_dir="responses/",                # evaluation set
    calibration_data_path="human_labels.jsonl",  # calibration source
)

print(results.metadata["oracle_sources"])
# {"calibration_data": {"n_oracle": 500, "coverage": 1.0},
#  "logged_data": {"n_oracle": 0, "coverage": 0.0},
#  "fresh_draws": {"n_oracle": 10, "coverage": None},
#  "total_oracle": 510, "n_conflicts": 0}
```

By default (`combine_oracle_sources=True`) oracle labels from the calibration file and the fresh draws are pooled — labels attach to responses, so every (judge, oracle) pair enters calibration and only true duplicates are deduped; cross-source disagreements on a prompt (>0.05) are warned about and recorded in `oracle_sources["conflicts"]`. Set `combine_oracle_sources=False` to calibrate from the file alone.

## Command Line Interface

```bash
# Analyze a fresh-draws directory (or a single JSONL with a target_policy field per record)
cje analyze responses/

# With a calibration file (old logged data works here)
cje analyze responses/ --calibration-data logged.jsonl

# Save results
cje analyze responses/ -o results.json

# Validate first
cje validate responses/
```

Real output on the bundled sample (`examples/arena_sample`):

```

==================================================

Results:
----------------------------------------
  base: 0.756 ± 0.010 (1 SE; 95% CI ≈ ±1.96·SE)
  clone: 0.763 ± 0.010 (1 SE; 95% CI ≈ ±1.96·SE)
```

The trophy is **reliability-aware**: if the argmax policy was flagged by the refusal gates (e.g. a REFUSE-LEVEL coverage badge), it is demoted to "⚠️ Best by point estimate: X (UNRELIABLE — see diagnostics)" and the best *reliable* policy is announced instead.

### `cje analyze PATH`

- `PATH`: fresh-draws directory of per-policy files, **or** a single JSONL file whose records carry a `target_policy` field
- `--estimator`: `calibrated-direct` (default) or `direct` — aliases of the same estimator; only `metadata["estimator"]` differs
- `--calibration-data FILE`: JSONL with judge + oracle pairs for learning the calibration
- `--estimator-config JSON`: e.g. `'{"n_bootstrap": 4000}'`
- `--judge-field` / `--oracle-field`: field names (defaults `judge_score` / `oracle_label`)
- `-o/--output FILE`: save results JSON
- `--fresh-draws-dir DIR`: alias for `PATH` (kept for 0.3.x compatibility)
- `-v/--verbose`, `-q/--quiet`

Logprob fields in the input earn an INFO note ("logprob fields present and ignored (Direct mode)") and nothing else.

### `cje validate PATH`

Checks that data is ready for `cje analyze` — validates the **raw parsed JSONL** (no model round-trip), reports per-policy sample and oracle counts, and exits 0 on valid data. `PATH` may be a fresh-draws directory, a single fresh-draws JSONL, or a calibration file (judge + oracle pairs without `target_policy` validate as a calibration source, with the exact `--calibration-data` invocation to use next).

## API Reference

### `analyze_dataset()`

```python
def analyze_dataset(
    logged_data_path=None,        # REMOVED in 0.4.0 — passing it raises the migration error
    fresh_draws_dir=None,         # directory of per-policy response files
    fresh_draws_data=None,        # in-memory alternative: {policy: [records]}
    calibration_data_path=None,   # JSONL with judge + oracle pairs
    combine_oracle_sources=True,  # pool oracle labels from all sources
    estimator="auto",             # "auto" -> "direct"; OPE names raise migration errors
    judge_field="judge_score",
    oracle_field="oracle_label",
    calibration_covariates=None,  # metadata fields for two-stage calibration
    include_response_length=False,  # auto word-count covariate (needs "response")
    estimator_config=None,        # kwargs forwarded to CalibratedDirectEstimator
    verbose=False,
) -> EstimationResult: ...
```

Provide `fresh_draws_dir` **or** `fresh_draws_data`.

**Returns** `EstimationResult` with:
- `.estimates` / `.standard_errors`: numpy arrays (order = `metadata["target_policies"]`)
- `.ci()` / `.confidence_interval()`: percentile bootstrap CIs by default; t-based with finite-sample df under `cluster_robust` (`.ci_info` records which; df metadata only exists there)
- `.compare_policies(i, j)`: paired policy comparison
- `.summary()`: compact text report (per-policy estimate + 95% CI + gate flags, best-policy line)
- `.best_policy()`: gate-aware `PolicyVerdict` (name, index, estimate, flagged, all_flagged, runner_up) — a flagged argmax is demoted to `runner_up` and the best reliable policy wins
- `.gates`: `Dict[str, GateResult]` — typed view of `metadata["reliability_gates"]`
- `.diagnostics`: `DirectDiagnostics` (statuses, boundary cards, calibration quality)
- `.calibrator`: fitted calibrator, reusable for transport audits
- `.metadata`: `target_policies`, `boundary_cards`, `reliability_gates`, `oracle_sources` (with `calibration_data_path`), `normalization`, `inference`, ...

The `estimator` names `"auto"`, `"direct"`, and `"calibrated-direct"` are aliases of the one calibrated estimator — whether calibration runs is driven solely by oracle-label availability (0 labels → the loud `naive_direct` fallback), and the parameter's only observable effect is which name is recorded in `metadata["estimator"]` (`"auto"` records `"direct"`).

### Removed in 0.4.0 (migration errors)

- `analyze_dataset(logged_data_path=...)` raises a `ValueError` explaining that OPE was removed and how to migrate — logged data with `judge_score` + `oracle_label` still works via `calibration_data_path`; IPS/DR users pin `pip install "cje-eval==0.3.*"`.
- `estimator="calibrated-ips"` / `"raw-ips"` / `"dr-cpo"` / `"mrdr"` / `"tmle"` / `"stacked-dr"` raise the same guidance (CLI and API share one validator in `interface/_removed.py`).

## Advanced Usage

### Inference methods

```python
# Default: cluster bootstrap with calibrator refit + augmented estimator
results = analyze_dataset(fresh_draws_dir="responses/")

# Equivalent explicit config
results = analyze_dataset(
    fresh_draws_dir="responses/",
    estimator_config={
        "inference_method": "bootstrap",   # "bootstrap" | "cluster_robust" | "auto"
        "use_augmented_estimator": True,
        "n_bootstrap": 2000,
    },
)

# Cluster-robust (fast; CRV1 SEs + oracle-jackknife calibration variance)
results = analyze_dataset(
    fresh_draws_dir="responses/",
    estimator_config={"inference_method": "cluster_robust"},
)
```

- **`bootstrap`** (default): cluster bootstrap with per-replicate calibrator refit and an AIPW-style augmented estimate — captures calibrator uncertainty and calibration/evaluation coupling. Recommended for valid CIs.
- **`cluster_robust`**: CRV1 cluster-robust SEs augmented with the delete-one-oracle-fold jackknife (`oua_jackknife=True` by default). Faster; the estimator falls back to this automatically (loudly, recorded in `metadata["inference"]`) when the evaluation draws carry no oracle labels — in that case it is exact, not an approximation.
- **`auto`**: cluster-robust, switching to bootstrap when calibration/evaluation coupling or few clusters are detected.

### Covariates

```python
# Length bias: auto-computed word count (all samples need a "response" field)
results = analyze_dataset(fresh_draws_dir="responses/", include_response_length=True)

# Custom metadata covariates (fields must exist in each record's metadata)
results = analyze_dataset(
    fresh_draws_dir="responses/",
    calibration_covariates=["domain", "difficulty"],
)
```

Covariates trigger two-stage calibration (see [`cje/calibration/README.md`](../calibration/README.md)).

### Transport audits

`analyze_dataset` exposes the fitted calibrator; audit it on a small oracle-labeled probe slice per deployment-relevant policy:

```python
import json
from cje.diagnostics import audit_transportability

probe = [json.loads(line) for line in open("probes/new_policy.jsonl")]
diag = audit_transportability(results.calibrator, probe, group_label="policy:new")
print(diag.summary())
# Transport: PASS/WARN/FAIL | Group: ... | N=... | δ̂: ... | Action: ...
```

See [`cje/diagnostics/README.md`](../diagnostics/README.md) and the [PLAYBOOK](../../PLAYBOOK.md) for the decision protocol.

## Summary

1. Generate fresh draws per policy, judge everything, oracle-label 5–25%.
2. `cje validate`, then `analyze_dataset(fresh_draws_dir=...)` (or the CLI).
3. Old logged data → `calibration_data_path`. IPS/DR → `pip install "cje-eval==0.3.*"`.
4. Trust the gates: a flagged policy's numbers are flagged for a reason.
